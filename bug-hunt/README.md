# Bug Hunt — интерактивное QA-задание

Демо-режим с **намеренно заложенными багами** в TaskFlow.

## Онлайн

**https://flowmolliface.github.io/qa/todo-app/?hunt=1**

## Локально

```bash
cd ../todo-app && npm run dev
```

**http://localhost:5173/?hunt=1**

## Задача

Найти **4 бага** без автотестов — только ручное исследование. Отмечайте найденное в панели справа.

## Для интервьюера

<details>
<summary>Спойлер: список багов (не открывать кандидату)</summary>

| ID | Баг | Как воспроизвести |
|----|-----|-------------------|
| BH-01 | Фильтр «Готовые» показывает все задачи | 2 задачи, одну отметить done → фильтр «Готовые» |
| BH-02 | Счётчик показывает общее число вместо активных | 2 active + 1 done → в футере неверная цифра |
| BH-03 | Пробелы проходят валидацию | Ввести `   ` и добавить задачу |
| BH-04 | Checkbox переключает предыдущую задачу | 3 задачи → отметить 2-ю или 3-ю → меняется соседняя |

</details>

## Что это показывает

- Exploratory testing
- Оформление багов (см. [bug-report-samples](../bug-report-samples/))
- Понимание UI-логики с двух сторон: dev и QA

Обычный режим (`/` без `?hunt=1`) — без багов, E2E-тесты зелёные.
